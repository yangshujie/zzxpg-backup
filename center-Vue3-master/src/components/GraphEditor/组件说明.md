GraphEditor 组件库文件作用详解
🎯 入口文件
index.js - 组件库入口
      
      作用：统一导出所有组件、工具函数、配置，供外部使用
      职责：
      1. 导出 Vue 组件（GraphContainer、LeftPanel、Toolbar、PropertyPanel）
      2. 导出组合式函数（useGraph、useGraphEvents等）
      3. 导出配置（scenes、nodes、edges）
      4. 导出工具类（DataConverter、eventBus）
      5. 提供 Vue 插件安装方法
index.css - 全局样式
css
      作用：定义图编辑器的全局样式
      内容包括：
      1. 容器布局样式（flex、定位）
      2. X6 画布样式（网格背景、节点悬浮效果）
      3. G6 画布样式
      4. 动画效果（淡入淡出）
      5. 响应式适配（移动端）
      6. 打印样式
🧩 Vue 组件
GraphContainer.vue - 主容器组件
      
      作用：图编辑器的核心容器，整合所有子组件
      职责：
      1. 管理画布生命周期（初始化、销毁）
      2. 协调左侧面板、工具栏、属性面板的显示
      3. 处理数据流（导入/导出）
      4. 监听用户交互事件（点击、拖拽、连线）
      5. 暴露方法给父组件（exportData、importData、zoom等）
      
      核心功能：
      - 根据 scene 自动选择 X6 或 G6
      - 支持 edit/view 模式切换
      - 支持只读模式
      - 统一的事件分发
LeftPanel.vue - 左侧面板组件
      
      作用：提供节点库和模板库
      职责：
      1. 树形结构展示节点分类
      2. 拖拽面板展示可拖拽的节点模板
      3. 连线库展示可用的连线类型
      4. 处理节点拖拽开始事件
      5. 处理节点选择事件
      
      三种展示模式：
      - 树形面板：层级结构的节点树
      - 拖拽面板：可拖拽的节点卡片
      - 连线库：展示各种连线样式和图例
Toolbar.vue - 工具栏组件
      
      作用：提供快捷操作按钮
      职责：
      1. 文件操作（新建、打开、保存、导出）
      2. 编辑操作（撤销、重做）
      3. 视图操作（缩放、适应视图）
      4. 工具选择（选择、拖拽、连线模式）
      5. 显示状态（缩放比例、保存状态）
      
      特点：
      - 根据场景配置动态显示按钮
      - 根据编辑状态禁用/启用按钮
PropertyPanel.vue - 属性面板组件
      
      作用：编辑选中节点的属性
      职责：
      1. 显示选中节点的基本信息（ID、类型）
      2. 编辑节点属性（标签、位置、大小）
      3. 根据节点类型动态显示自定义属性
      4. 处理属性更新
      5. 提供删除节点功能
      
      支持的属性类型：
      - 基础属性：标签、坐标
      - 节点特有属性：如指标数值、算法参数、人员信息
⚙️ 核心引擎层 (core/)
GraphEngine.js - 图引擎管理器（单例）
      
      作用：管理图引擎实例的创建和切换
      职责：
      1. 创建 X6 或 G6 适配器实例
      2. 管理当前激活的适配器
      3. 提供引擎切换功能
      4. 统一销毁和清理资源
      
      设计模式：单例模式
      确保整个应用只有一个图引擎管理器
adapters/BaseAdapter.js - 适配器基类
      
      作用：定义统一的图引擎接口规范
      职责：
      1. 定义所有图引擎必须实现的方法
      2. 提供通用的节点/边管理逻辑
      3. 集成注册中心，自动获取节点配置
      4. 统一事件处理机制
      
      定义的核心方法：
      - init/destroy: 生命周期管理
      - addNode/addEdge: 添加元素
      - removeNode/removeEdge: 删除元素
      - updateNode/updateEdge: 更新元素
      - importData/exportData: 数据导入导出
      - zoomTo/fitView: 视图控制
      - on/off: 事件监听
adapters/X6Adapter.js - X6适配器
      
      作用：封装 AntV X6 的具体实现
      职责：
      1. 初始化 X6 Graph 实例
      2. 实现 X6 特有的节点注册（支持 HTML/React/Vue）
      3. 实现 X6 特有的功能（历史记录、键盘快捷键、网格对齐）
      4. 处理 X6 的事件并转换为统一事件格式
      5. 实现 X6 的数据序列化/反序列化
      
      X6 特性支持：
      - 撤销/重做历史
      - 键盘快捷键
      - 网格对齐线
      - 连线验证
      - 端口连接
adapters/G6Adapter.js - G6适配器
      
      作用：封装 AntV G6 的具体实现
      职责：
      1. 初始化 G6 Graph 实例
      2. 实现 G6 特有的节点注册
      3. 实现 G6 特有的布局算法（力导向、树形等）
      4. 支持组合节点（Combo）
      5. 支持路径高亮、节点状态管理
      
      G6 特性支持：
      - 多种布局算法
      - 组合节点折叠/展开
      - 节点状态管理
      - 大规模数据渲染
registry/NodeRegistry.js - 节点类型注册中心
      
      作用：统一管理所有节点类型配置
      职责：
      1. 注册节点类型（支持 X6 和 G6 双配置）
      2. 提供节点配置查询
      3. 管理节点多模式样式（default、highlight、selected）
      4. 提供节点模板获取
      
      使用场景：
      - 左侧面板展示节点库
      - 添加节点时自动获取配置
      - 动态切换节点样式
registry/EdgeRegistry.js - 连线类型注册中心
      
      作用：统一管理所有连线类型配置
      职责：
      1. 注册连线类型（支持 X6 和 G6 双配置）
      2. 提供连线配置查询
      3. 管理连线样式（颜色、线型、箭头）
      
      支持的连线类型：
      - 隶属关系（实线箭头）
      - 指挥关系（双实线）
      - 协作关系（虚线）
      - 影响关系（点线）
      - 组合关系（双向箭头）
registry/PluginRegistry.js - 插件注册中心
      
      作用：管理图编辑器的扩展插件
      职责：
      1. 注册插件
      2. 应用插件到图实例
      3. 管理插件生命周期
      
      插件示例：
      - 小地图插件
      - 水印插件
      - 导出图片插件
      - 键盘快捷键插件
🎣 组合式函数 (composables/)
useGraph.js - 图引擎核心Hook
      
      作用：封装图引擎的 Vue 组合式 API
      职责：
      1. 管理图实例的生命周期
      2. 提供响应式的图状态（loading、error）
      3. 封装图操作方法（addNode、addEdge等）
      4. 自动清理资源
      
      返回值：
      - adapter: 图适配器实例
      - loading: 加载状态
      - error: 错误信息
      - init: 初始化方法
      - destroy: 销毁方法
      - addNode/addEdge: 添加元素方法
useGraphEvents.js - 图事件Hook
      
      作用：管理图事件监听
      职责：
      1. 注册/注销事件监听
      2. 自动清理事件（组件卸载时）
      3. 事件转发
      
      支持的事件：
      - node:click: 节点点击
      - edge:click: 边点击
      - node:moved: 节点移动
      - edge:connected: 连线完成
useGraphTools.js - 工具Hook
      
      作用：管理工具栏状态和操作
      职责：
      1. 管理当前激活的工具（选择、拖拽、连线）
      2. 管理视图状态（缩放、位置）
      3. 提供工具操作方法（放大、缩小、撤销等）
      
      工具方法：
      - setTool: 切换工具
      - zoomIn/zoomOut: 缩放
      - undo/redo: 撤销重做
      - exportData/importData: 数据导入导出
useSceneConfig.js - 场景配置Hook
      
      作用：管理场景配置
      职责：
      1. 获取当前场景配置
      2. 判断引擎类型（X6/G6）
      3. 获取节点类型列表
      4. 获取工具栏配置
      
      返回值：
      - sceneConfig: 场景完整配置
      - engineType: 引擎类型
      - isEditable: 是否可编辑
      - nodeTypes: 节点类型列表
      - toolbarConfig: 工具栏配置
⚙️ 配置层 (config/)
scenes.js - 场景配置管理
      
      作用：定义所有场景的配置
      职责：
      1. 配置每个场景使用的引擎（X6/G6/双模式）
      2. 配置场景的默认参数（网格、布局等）
      3. 定义场景支持的节点类型和连线类型
      4. 定义场景的工具栏按钮
      
      场景类型：
      - flowChart: 流程图（X6编辑）
      - knowledgeGraph: 知识图谱（G6展示）
      - umlDiagram: UML图（X6编辑）
      - socialNetwork: 社交网络（G6展示）
      - dualMode: 双模式（X6编辑+G6展示）
nodes.js - 节点类型配置
      
      作用：定义所有节点类型的配置
      职责：
      1. 定义节点的 X6 样式配置
      2. 定义节点的 G6 样式配置
      3. 定义节点的多模式样式（高亮、选中）
      4. 定义节点的图标和描述
      5. 注册节点到注册中心
      
      节点分类：
      - 指标体系节点：indicator_root, indicator_level1, indicator_level2
      - 算法节点：data_source, algorithm, output, preprocess
      - 结构节点：organization, department, person, team, role
edges.js - 连线类型配置
      
      作用：定义所有连线类型的配置
      职责：
      1. 定义连线的 X6 样式配置
      2. 定义连线的 G6 样式配置
      3. 定义连线的颜色、线型、箭头样式
      4. 注册连线到注册中心
      
      连线类型：
      - belong_to: 隶属关系（实线箭头）
      - command: 指挥关系（双实线）
      - cooperate: 协作关系（虚线）
      - influence: 影响关系（点线）
      - compose: 组合关系（双向箭头）
🛠️ 工具函数 (utils/)
dataConverter.js - 数据格式转换
      
      作用：在不同数据格式之间转换
      职责：
      1. X6 数据 ↔ 统一格式
      2. G6 数据 ↔ 统一格式
      3. 统一格式 → X6 展示格式
      4. 统一格式 → G6 展示格式
      5. 数据验证
      
      为什么需要：
      - X6 和 G6 的数据结构不同
      - 需要在编辑和展示模式间切换
      - 统一数据格式便于存储和传输
eventBus.js - 事件总线
      
      作用：组件间通信
      职责：
      1. 提供事件订阅/发布机制
      2. 支持一次性事件
      3. 支持事件取消订阅
      4. 跨组件通信
      
      使用场景：
      - 工具栏触发保存，通知主容器
      - 属性面板更新节点，通知画布
      - 左侧面板拖拽节点，通知画布添加
📦 状态管理 (store/modules/)
graph.js - 图状态模块
      
      作用：管理全局图状态
      职责：
      1. 存储当前场景配置
      2. 存储图数据（节点、边）
      3. 存储选中状态
      4. 管理历史记录（撤销/重做）
      5. 管理加载状态和修改状态
      
      State 结构：
      - currentScene: 当前场景
      - graphData: 图数据
      - selected: 选中元素
      - history: 历史记录栈
      - loading: 加载状态
      - isModified: 是否已修改
🌐 API 接口层 (api/)
graph.js - 图数据接口
      
      作用：封装后端 API 调用
      职责：
      1. 获取图数据
      2. 保存图数据
      3. 导出图数据
      4. 获取节点模板库
      5. 获取图统计信息
      6. 图数据分享
      7. 版本历史管理
      
      API 列表：
      - getGraphData: 获取图数据
      - saveGraphData: 保存图数据
      - exportGraphData: 导出图数据
      - getNodeTemplates: 获取节点模板
      - getGraphStatistics: 获取统计信息
      - shareGraph: 分享图数据
      - getGraphHistory: 获取版本历史
