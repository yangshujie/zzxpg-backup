# 姿态控制精度算法优化版本

## 概述

这是姿态控制精度算法的优化版本，主要针对性能进行了全面优化，同时保持了与原始版本的接口兼容性。

## 主要优化点

### 1. 安全性优化
- **移除 eval() 调用**：使用 `int()` 和 `float()` 进行类型转换，避免代码注入风险
- **参数验证**：添加了更严格的参数检查和默认值处理

### 2. 性能优化

#### 2.1 向量化操作
- 使用 NumPy 向量化操作替代 Python 循环
- 批量处理数组操作，减少函数调用开销
- 使用广播机制进行批量计算

#### 2.2 连续段检测优化
- **原始方法**：使用正则表达式 `re.sub()` 处理数组转字符串
- **优化方法**：使用 NumPy 的 `diff()` 和布尔掩码直接检测连续段
- **性能提升**：对于大数据集，性能提升可达 10-100 倍

#### 2.3 索引计算优化
- 使用 `np.intersect1d()` 和 `np.setdiff1d()` 替代列表操作
- 减少重复的数组转换
- 优化内存分配，避免不必要的数组复制

#### 2.4 数据提取优化
- 一次性转换所有数据，避免重复转换
- 使用集合（set）进行快速成员检查
- 优化数据拼接操作

### 3. 代码结构优化
- 提取重复逻辑为独立函数
- 改进函数命名，提高可读性
- 添加性能监控（计算耗时）

## 性能对比

### 优化前（原始版本）
- 使用 `eval()` 进行类型转换（安全风险）
- 使用正则表达式检测连续段（性能瓶颈）
- 大量 Python 循环和列表操作
- 重复的数组转换

### 优化后（本版本）
- 类型安全的参数处理
- NumPy 向量化连续段检测
- 最小化循环，最大化向量化操作
- 减少内存分配和数组复制

### 预期性能提升
- **小数据集**（< 1000 点）：提升 2-5 倍
- **中等数据集**（1000-10000 点）：提升 5-20 倍
- **大数据集**（> 10000 点）：提升 10-100 倍

## 关键函数说明

### `calMain(datas, config)`
主计算函数，处理所有天的数据。

**优化点**：
- 预处理配置参数，避免重复转换
- 优化数据提取逻辑
- 使用向量化操作

### `get_y_ang_idx_optimized(y_ang, today_times, diffValue, count)`
优化的角度索引计算函数。

**优化点**：
- 减少嵌套循环
- 使用向量化操作进行差分计算
- 优化删除索引的逻辑

### `detect_continuous_segments_optimized(tmp_idx, data_len, count)`
优化的连续段检测函数。

**优化点**：
- 使用 NumPy 替代正则表达式
- 直接操作索引数组，避免字符串转换

### `get_flag_indices_optimized(y_control, flag_values)`
优化的标志索引计算函数。

**优化点**：
- 使用广播机制批量计算
- 使用逻辑或合并掩码
